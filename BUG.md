最值得补充的功能

2. 任务依赖关系（前置/后置任务）

Kanban 视图中任务之间是平级的，无法设置依赖。这会导致无法识别关键路径。可以引入 task_dependencies 表，支持"阻塞/被阻塞"关系可视化。

3. 标签系统（Tags）

现在只有优先级的维度来分类任务。缺少灵活的标签系统进行跨项目的任务组织、筛选。看 migrations 里也没有 tags 表。

4. 数据导入/导出

没有导出 JSON/CSV、备份 SQLite 的功能，也没有从其他工具（Linear、Jira）导入的能力。一个桌面工具没有导出机制数据风险较高。

6. 报表和可视化

Dashboard 的统计目前较基础。缺少：燃尽图（Burndown Chart）、任务完成趋势（按周/月）、各成员工作量对比、项目健康度概览。

7. 回收站

删除的任务/项目目前是物理删除。软删除 + 回收站可以在误删时挽回数据。
